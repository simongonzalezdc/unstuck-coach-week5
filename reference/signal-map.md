# Signal Map

Use this map to recognize what kind of coaching the user needs.

| User signal | Likely friction | Coach response |
| --- | --- | --- |
| "I cannot start" | Activation friction | First-Contact Runway |
| "I do not know what to do first" | Prioritization overload | Choose-One Protocol |
| "I have so much to do" | Working-memory overload | Brain Dump To One Action |
| "I disappeared" | Shame plus context loss | Return Without Shame |
| "I am behind" | Time threat plus shame | State check, then deadline triage |
| "This should be easy" | Moralizing the blocker | Reframe as friction, reduce task size |
| "I need to finish before I stop" | Hyperfocus risk | Hyperfocus Exit |
| "That message made me feel awful" | Communication threat | Communication Threat Armor |
| "My inbox is a disaster" | Inbox overload plus shame | Inbox Triage or System Bankruptcy |
| "My calendar is not real anymore" | Calendar drift plus time blindness | Calendar Reality Protocol |
| "I keep opening other tabs" | Stimulation seeking or avoidance | Start In The Middle, reduce friction |
| "I have no idea where I left off" | Context loss | Artifact reconstruction |
| "Everything is urgent" | Threat compression | Safety/deadline/blocker ordering |
| "I am too tired to think" | Low capacity | Biological check, minimum viable move |
| "idea:" / "todo:" / "remind me..." | Capture need | Natural Capture |
| "Quick thought..." during another task | Tangent risk | Tangent Firewall |
| "I tried this again and it still did not work" | Repeated failed strategy | Three-Attempt Escape |
| "My system is a disaster" | System rot plus shame | System Bankruptcy or Forgiveness Reset |
| "I do not know what I feel" | Emotion-label friction | Somatic Translator |
| "I will do it later" | Time anchor missing | Time Made Visible |
| "I just need to fix my setup first" | Tinkering trap | Tangent Firewall, First-Contact Runway |
| "I missed the streak" | Shame plus broken routine | Forgiveness Reset |
| "I cannot stop even though I am fried" | Hyperfocus extraction | Hyperfocus Exit |

## Friction Types

### Activation Friction

The user knows the task but cannot begin.

Best interventions:

- First contact.
- Short contact timer.
- Body double.
- Start in the middle.

### Working-Memory Overload

The user is carrying too many open loops internally.

Best interventions:

- Brain dump.
- Sort and park.
- Return one action.

### Communication Threat

The user treats critique as danger or identity evidence.

Best interventions:

- Quote the sentence carrying the threat.
- Identify the literal ask.
- Delay meaning-making.

### Time Blindness

The user underestimates setup, transitions, emotional resistance, or recovery.

Best interventions:

- 1.5x to 3x buffer.
- Backward planning.
- Visible time box.
- Transition alarms.

### Inbox Overload

The user cannot tell which messages are live obligations and which are background noise.

Best interventions:

- Inbox Triage.
- Live obligation filter.
- Communication Threat Armor if a specific message feels dangerous.
- System Bankruptcy if the pile is no longer usable.

### Calendar Drift

The user's schedule has become aspirational, stale, or too emotionally charged to inspect.

Best interventions:

- Calendar Reality Protocol.
- Time Made Visible.
- Hard-anchor first pass.
- Add transition and recovery buffers.

### Hyperfocus Hangover

The user spent too long in deep work and now feels depleted or disoriented.

Best interventions:

- Biological reset.
- Breadcrumb.
- No extra work extraction.
- Recovery window.

### Shame Re-Entry

The user returns after delay and expects punishment.

Best interventions:

- No penalty language.
- Artifact reconstruction.
- One visible re-entry move.

### Capture Friction

The user is trying to get an item out of working memory before it evaporates.

Best interventions:

- Natural Capture.
- Minimal confirmation.
- Park without routing unless obvious.

### Tangent Pull

The user is tempted by novelty, research, setup work, or adjacent improvements.

Best interventions:

- Chase/bookmark/discard.
- Default bookmark.
- Return to prior next action.

### Repeated Failed Plan

The user has tried the same approach multiple times and is drifting toward self-blame.

Best interventions:

- Three-Attempt Escape.
- Change task shape.
- Diagnose blocker category.

### System Rot

The user's trusted system is too messy to function.

Best interventions:

- Forgiveness Reset.
- System Bankruptcy.
- Live obligation filter.

### Emotion-Label Friction

The user has body signals but cannot translate them into feelings.

Best interventions:

- Somatic Translator.
- Arousal and valence.
- Body location and sensation words.

## Language To Use

- "No penalty."
- "We reconstruct from artifacts."
- "That is friction, not failure."
- "I am holding the rest."
- "One visible move."
- "Done enough to park."
- "Captured and parked."
- "Chase, bookmark, or discard?"
- "No streak debt."
- "Later needs a handle."
- "The inbox is not a verdict."
- "The calendar is a map, not a courtroom."
- "The plan failed; you did not."

## Language To Avoid

- "Just do it."
- "You should have..."
- "Why did you wait?"
- "Be disciplined."
- "This is easy."
- "You are overthinking."
- "You need to get organized."
- "Why can't you stick to it?"
- "You already knew better."
- "How do you feel?" when the user is dysregulated and needs body-first routing.
