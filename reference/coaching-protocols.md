# Coaching Protocols

Use these protocols as reusable coaching moves. Do not dump them all at once. Pick the one that fits the user's state.

## Protocol: Traffic Light Check

Use when the user sounds overwhelmed, frozen, ashamed, anxious, or scattered.

Question:

> Green, yellow, or red right now?

Interpretation:

- Green: planning is available.
- Yellow: regulate first, then choose.
- Red: no task pressure; gentle grounding only.

Next moves:

- Green: "What result are we moving?"
- Yellow: "Feet on floor. One breath. What is the loudest task?"
- Red: "No rush. Can you wiggle your toes or name one object in the room?"

## Protocol: Somatic Translator

Use when the user cannot name an emotion, sounds dysregulated, or says "I feel weird/off/bad."

Do not start with "How do you feel?" Start with body data.

Questions:

- "High energy or low energy?"
- "Pleasant, unpleasant, or mixed?"
- "Where is it loudest: head, chest, stomach, jaw, hands, legs?"
- "Is it tight, hot, buzzy, heavy, empty, static, or something else?"

Then translate to a coaching move:

| Body signal | Possible coaching move |
| --- | --- |
| High energy + unpleasant | Ground, reduce inputs, choose one outlet |
| Low energy + unpleasant | Lower task pressure, tiny body cue, recovery first |
| High energy + pleasant | Channel into one bounded sprint |
| Low energy + neutral | Minimum viable move or clean park |

Script:

> You do not need the right emotion word. Give me the body signal and I will route from there.

## Protocol: First-Contact Runway

Use when the user knows the task but cannot initiate.

Script:

> We are not doing the whole task. We are creating one visible point of contact.

Steps:

1. Open the artifact.
2. Touch one harmless part.
3. Leave a marker.
4. Report what changed.

Examples of first contact:

- Put the form on the table.
- Open the message without replying.
- Put the dish in the sink.
- Set the bill beside the keyboard.
- Put shoes by the door.
- Add a scratch note.
- Rename a scratch note.
- Open the relevant tab or page.
- Paste a confusing sentence into a note.

## Protocol: Natural Capture

Use when the user drops a raw item with a capture phrase.

Capture phrases:

- "idea:"
- "todo:"
- "note to self:"
- "remind me to..."
- "bookmark this"
- "brain dump:"
- "don't let me forget..."

Steps:

1. Repeat a cleaned version of the captured item.
2. Put it in Now, Later, Waiting, or Reference if obvious.
3. Ask whether to route now or park.

Script:

> Captured: [item]. I parked it in [bucket]. Back to [prior next action]?

Rule:

Capture is not a request for a lecture. Keep the confirmation tiny.

## Protocol: Start In The Middle

Use when the beginning feels blocked.

Questions:

- "What part of this is least annoying?"
- "What piece would be fun if nobody saw it?"
- "What do you already know how to do?"

Rule:

Starting in the middle counts. Sequence can be repaired later.

## Protocol: Brain Dump To One Action

Use when the user lists many tasks or feels mentally crowded.

Steps:

1. Invite an unstructured dump.
2. Sort into Now, Later, Waiting, Trash.
3. Return only the Now item.
4. Keep the rest parked.

Output format:

```text
Now: [one item]
Later: [held list]
Waiting: [blocked items]
Trash: [items to ignore]

Next action: [single visible move]
```

## Protocol: System Bankruptcy

Use when the user has an inbox, task list, tab set, notes system, admin pile, or physical mess so noisy that cleanup itself has become the blocker.

Steps:

1. Validate that the pile is no longer a useful working surface.
2. Archive the pile without processing every item.
3. Rescue only live obligations.
4. Create one clean restart surface.

Script:

> We are declaring system bankruptcy. The goal is not to process the whole pile. The goal is to restore a surface you can act from.

Live obligation filter:

- Due this week.
- Blocks another person.
- Has money, safety, legal, or relationship consequences.
- Already scheduled.
- Truly still wanted.

Everything else can move to an archive.

## Protocol: Communication Threat Armor

Use when a message, critique, comment, reply, or other communication triggers shame or threat.

Steps:

1. Separate the person from the message.
2. Quote the sentence carrying the threat.
3. Sort literal ask, feeling, consequence, and next reply.
4. Answer the ask only after it is visible.
5. Give the feeling care without letting it run the plan.

Buckets:

- Literal ask: what the other person is actually requesting.
- Feeling: what the message activated.
- Consequence: money, safety, legal, relationship, or schedule impact.
- Next reply: the smallest useful response.

Script:

> We are not processing your worth through a notification. Quote the sentence that carries the threat.

## Protocol: Tangent Firewall

Use when the user is tempted by a side quest, research rabbit hole, new tool, redesign, reorganization, or "quick" cleanup that is not the current task.

Ask:

> Chase, bookmark, or discard?

Definitions:

- Chase: make the tangent the task on purpose.
- Bookmark: capture it and return to the original task.
- Discard: let it go.

Default:

If the user cannot choose, bookmark and return.

Script:

> Bookmarked. The prior next action is still: [action].

## Protocol: Time Made Visible

Use when the user has a deadline, says "later," underestimates a task, or is drifting.

Steps:

1. Name the hard stop or desired stop.
2. Work backward.
3. Add setup, transition, and re-entry time.
4. Choose a visible anchor.

Visible anchors:

- Clock time.
- Countdown timer.
- Calendar block.
- Song or playlist length.
- Meeting boundary.
- Checkpoint count.

Script:

> Later needs a handle. Is this 3:30, after the call, or after one visible contact pass?

Estimate rule:

- Familiar and emotionally neutral: multiply by 1.5.
- Familiar but sticky: multiply by 2.
- Novel, ambiguous, or shame-loaded: multiply by 3.

## Protocol: Hyperfocus Exit

Use when the user needs to stop, transition, or recover.

Steps:

1. Biological reset: water, food, bathroom, movement.
2. Externalize the current state.
3. Leave a breadcrumb.
4. Stop on purpose.

Breadcrumb format:

```text
Next time, start with:
- command:
- file:
- question:
```

If the user is depleted:

> No evaluation yet. Biological reset first, then breadcrumb, then stop.

## Protocol: Return Without Shame

Use when the user has been absent, avoidant, late, or stuck.

Opening:

> Welcome back. No penalty. We reconstruct from artifacts.

Steps:

1. Find the last real artifact.
2. Rebuild current state.
3. Choose one re-entry move.
4. Park the moral story.

Artifacts:

- Branch.
- File.
- Email.
- Calendar item.
- Note.
- Issue.
- Command output.

## Protocol: Three-Attempt Escape

Use when the user has tried the same approach repeatedly without traction.

Steps:

1. Count the attempts.
2. Name the loop.
3. Stop the repeated strategy.
4. Diagnose the blocker category.
5. Offer a different shape.

Blocker categories:

- Energy.
- Fear or shame.
- Missing context.
- Too many choices.
- Task too large.
- Environment too noisy.

Script:

> This has failed three times, so the plan is the problem now. We are changing the shape.

## Protocol: Forgiveness Reset

Use when the user missed a streak, broke a routine, abandoned a plan, or feels guilty about system rot.

Steps:

1. Remove moral debt.
2. Keep any useful data.
3. Restart smaller.
4. Make the next return easier.

Script:

> No streak debt. Missed days are data. What is the smallest restart that would still count?

## Protocol: Finish Or Park

Use when work is ending.

Ask:

> Is this done, delegated, scheduled, or parked?

Then capture:

- What moved.
- What remains.
- Where to restart.

The goal is re-entry, not perfect closure.

## Protocol: Tiny Accountability

Use when the user needs a light check-in.

Prompt:

> Tell me when one surface is open, touched, or visible.

or:

> Come back with the number of comments, not the answers.

Make the proof small, visible, and hard to fake accidentally.
